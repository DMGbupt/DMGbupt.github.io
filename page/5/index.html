CSS的浮动笔记

发表于 2016-04-04

标准流中，块级元素总是独占一行。而复杂布局中，最基本的一个问题就是如何一行显示多个块级元素。显然标准流无法满足要求，所以我们才引入浮动。
浮动，正如其名字一样，使块级元素脱离标准流，“漂浮”在标准流之上，标准流中不再保存该元素位置，和标准流不在一个层次。

定义浮动

我们通过定义float属性值来控制浮动。float默认是none，可以设置left和right。
如果浮动元素的前一个元素仍在标准流中，那么浮动元素相对垂直位置不会改变，也就是说浮动的顶部总是和上一个元素的底部对齐，左右则根据浮动设置相应贴边；如果前一个也是浮动元素，那就紧跟在前一个元素后边（根据浮动的设置，“后边”的实际显示位置可能是右边或左边），一行放不下则挤到下一行。
但是，如果浮动元素后面的元素是在标准流中的，那么后面的元素会紧跟浮动元素（一行放不下则挤到下一行）。如果我们想让后面元素从新的一行按正常流显示，则需要清除浮动。

清除浮动

清除浮动可以理解为通过下移来打破浮动元素的横向排列规则（实际是增加了清除浮动元素的上边的空间）。
对浮动元素设置clear属性：

  • none : 默认值，允许元素两边都可以有浮动对象
  • left : 操作元素使其左边没有浮动对象
  • right : 操作元素使其右边没有浮动对象
  • both : 操作元素使其两边没有浮动对象
    这个clear属性只能影响使用的元素本身，不能操作其他元素。例如两个左浮动的元素，想让第二个元素下移一行，则需要对第二个元素设置clear：left，使其下移一行来满足左边没浮动对象。如果对第一个元素使用clear：right，似乎按照字面来说能清除第一个元素右边的浮动元素，其实不然，因为clear：right仅对第一个元素起作用，而第一个元素不论怎么下移都不会使二个元素相对下移。
    所以对左浮动的元素使用clear：left，对右浮动的元素使用clear：right。

浮动问题总结

浮动元素引起的问题：

  • 父元素的高度无法被撑开，影响与父元素同级的元素
  • 与浮动元素同级的非浮动元素会跟随其后
  • 若非第一个元素浮动，则该元素之前的元素也需要浮动，否则会影响页面显示的结构

C++类中的const

发表于 2016-02-12

1.const修饰成员变量
const修饰类的成员变量，表示成员是常量，不能被修改，同时它只能在初始化列表中赋值。

2.const修饰成员函数
const修饰类的成员函数，则该成员函数不能修改类中任何成员。一般写在函数的最后来修饰。

如果非要在const修饰的函数里改变某成员变量，则只需用mutable修饰该成员变量即可。

3.const修饰类对象/对象指针/对象引用
const修饰类对象表示该对象为常量对象，其中的任何成员都不能被修改，且该对象的任何非const成员函数都不能被调用，因为任何非const成员函数会有修改成员变量的企图。对于对象指针和对象引用也是一样。

C++强制类型转换（cast）

发表于 2016-02-12

dynamic_cast: 通常在基类和派生类之间转换时使用，将一个基类对象指针（或引用）cast到继承类指针（通常要求基类有虚函数），dynamic_cast会根据基类指针是否真正指向继承类指针来做相应处理。
const_cast: 编译器在编译期处理，去掉类型中的const属性，主要针对const以及volatile属性。
static_cast: 一般的转换，不做运行时检查，需要自己确保安全。通常，有明确意义的转换（不涉及底层const），都可以用static_cast。

JS中所有对象都继承自Object对象，所以不论是普通对象还是函数对象都会继承一些公共的属性和方法，这里记录下比较有用的几个。

可用于所有对象实例的属性和方法

  • constructor属性，指向当前对象的构造函数对象。

a和b都是Object()构造器创建的，所以它们的constructor都指向同一个构造函数对象。F是Function()构造的一个函数对象，它的constructor指向Function()函数对象。

  • hasOwnProperty(Name)方法，判断Name属性是否只存在当前对象中，而不在对象的原型链中。

arr是数组，拥有自己的属性length，但是hasOwnProperty方法却是从Object那继承的，所以是false。

  • hasOwnProperty和for…in结合，可以获取对象自己的所有属性。

  • propertyIsEnumerable(Name)方法，判断Name属性是否可以被for…in枚举。

说明对象的constructor是不可被枚举的。

  • isPrototypeOf(Obj)方法，判断对象是否在obj的原型链上。

foo由F()构造，所以F.prototype在foo的原型链上。

  • toString()方法，返回对象的字符串表示。

  • toLocaleString()方法，同样返回对象的字符串表示，跟所在地区有关。

  • valueOf()方法，返回对象的原始值，可能是字符串、数值或bool值等。
