C++强制类型转换（cast）

发表于 2016-04-12   |  

dynamic_cast: 通常在基类和派生类之间转换时使用，将一个基类对象指针（或引用）cast到继承类指针（通常要求基类有虚函数），dynamic_cast会根据基类指针是否真正指向继承类指针来做相应处理。上行时和static_cast作用相同，但下行时会做类型检查，比static_cast安全。
static_cast: 一般的转换，不做运行时检查，需要自己确保安全。通常，只要是有明确意义的转换（不涉及底层const），都可以用static_cast。
const_cast: 编译器在编译期处理，去掉类型中的const属性，主要针对const以及volatile属性。
reinterpret_cast：用来处理无关类型之间的转换。它会产生一个新的值，这个值会有与原始参数（expressoin）有完全相同的比特位。

http状态码

发表于 2016-04-11   |  

常用状态码

100 Continue 继续，一般在发送post请求时，已发送了http header之后服务端将返回此信息，表示确认，之后发送具体参数信息
200 OK 正常返回信息
201 Created 请求成功并且服务器创建了新的资源
202 Accepted 服务器已接受请求，但尚未处理
301 Moved Permanently 请求的网页已永久移动到新位置。
302 Found 临时性重定向。
303 See Other 临时性重定向，且总是使用 GET 请求新的 URI。
304 Not Modified 自从上次请求后，请求的网页未修改过。

400 Bad Request 服务器无法理解请求的格式，客户端不应当尝试再次使用相同的内容发起请求。
401 Unauthorized 请求未授权。
403 Forbidden 禁止访问。
404 Not Found 找不到如何与 URI 相匹配的资源。

500 Internal Server Error 最常见的服务器端错误。
503 Service Unavailable 服务器端暂时无法处理请求（可能是过载或维护）。

C++类中的const修饰符

发表于 2016-03-28   |  

1.const修饰成员变量
const修饰类的成员变量，表示成员是常量，不能被修改，同时它只能在初始化列表中赋值。

2.const修饰成员函数
const修饰类的成员函数，则该成员函数不能修改类中任何成员。一般写在函数的最后来修饰。

如果非要在const修饰的函数里改变某成员变量，则只需用mutable修饰该成员变量即可。

3.const修饰类对象/对象指针/对象引用
const修饰类对象表示该对象为常量对象，其中的任何成员都不能被修改，且该对象的任何非const成员函数都不能被调用，因为任何非const成员函数会有修改成员变量的企图。对于对象指针和对象引用也是一样。

数据库事务要点

发表于 2016-03-23   |  

ACID：指数据库事务正确执行的四个基本要素的缩写。包含：
原子性（Atomicity）：事务必须是原子工作单元；对于其数据修改，要么全都执行，要么全都不执行。
一致性（Consistency）:事务在完成时，必须使所有的数据都保持一致状态。
隔离性（Isolation）:事务查看数据时数据所处的状态，要么是另一并发事务修改它之前的状态，要么是另一事务修改它之后的状态，事务不会查看中间状态的数据。
持久性（Durability）：事务完成之后，它对于系统的影响是永久性的。

关于隔离性：
MySQL中事务隔离级别：Read Uncommitted，Read Committed，Repeatable Read（MySQL的默认事务隔离级别）和Serializable。
不同的隔离级别会产生不同的事务执行异常现象：脏读、不可重复读、幻读。
脏读（Dirty Read)：事务A读取了事务B未提交的数据。这个数据是有可能回滚的，因此在脏数据上所做的操作可能是不正确的。
不可重复读(Unrepeatable Read)：同一事务中，两次相同的查询却返回不同数据。这是由于第二次查询时读取了其他事务已提交的更新数据。
幻读(Phantom Read)：同一事务中，两次相同的操作，返回不同的记录数。这是事务不是独立执行时发生的一种现象，例如第一个事务对一个表中的数据进行了修改，这种修改涉及到表中的全部数据行。同时，第二个事务向表中插入一行新数据。那么，进行第一个事务的用户可能就会发现表中还有没有修改的数据行，就好象发生了幻觉一样。幻读与不可重复读的区别在于：幻读侧重于新增或删除数据的情况，而不可重复读侧重于修改数据的情况。
下面表格给出不同隔离级别允许发生的异常现象。
隔离级别 | 脏读 | 不可重复读 | 幻读
:—————-|——|————|—–
Read Uncommitted | 允许 | 允许 | 允许
Read Committed | 禁止 | 允许 | 允许
Repeatable Read | 禁止 | 禁止 | 允许
Serializable | 禁止 | 禁止 | 禁止
由于高度隔离会限制可并行执行的事务数，所以一些应用程序会降低隔离级别以换取更大的吞吐量。
