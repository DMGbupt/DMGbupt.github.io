大端和小端模式

发表于 2016-05-03

大于一个字节的结构（例如int）存储数据时，会产生不同字节序。
小端模式：内存低地址存数据的低位。
大端模式：内存低地址存数据的高位。

可以使用union判断处理器的大小端。

union中的成员都是从低地址开始存放。本例中union共分配了4个字节，存a成员时使用内存地址从低到高的4个字节，存b成员时使用内存地址最低的一个字节，所以可以从b的值来看出低地址端存的是什么值。
1的存储形式是0x00 00 00 01，所以数据的最低位是01，最高位是00。若是小端存储，则低地址端存01，若是大端存储，则低地址端存00。由此，b的值为1时是小端模式，为0时是大端模式。

C++中new和malloc区别

发表于 2016-05-03

1,malloc与free是C++/C语言的标准库函数，new/delete是C++的运算符。它们都可用于申请动态内存和释放内存。
2,对于非内部数据类型的对象而言，光用maloc/free无法满足动态对象的要求。对象在创建的同时要自动执行构造函数，对象在消亡之前要自动执行析构函数。由于malloc/free是库函数而不是运算符，不在编译器控制权限之内，不能够把执行构造函数和析构函数的任务强加于malloc/free。
3,因此C++语言需要一个能完成动态内存分配和初始化工作的运算符new，以及一个能完成清理与释放内存工作的运算符delete。注意new/delete不是库函数。
4,new可以认为是malloc加构造函数的执行。new出来的指针是直接带类型信息的。而malloc返回的都是void指针。

C++强制类型转换（cast）

发表于 2016-04-12

dynamic_cast: 通常在基类和派生类之间转换时使用，将一个基类对象指针（或引用）cast到继承类指针（通常要求基类有虚函数），dynamic_cast会根据基类指针是否真正指向继承类指针来做相应处理。上行时和static_cast作用相同，但下行时会做类型检查，比static_cast安全。
static_cast: 一般的转换，不做运行时检查，需要自己确保安全。通常，只要是有明确意义的转换（不涉及底层const），都可以用static_cast。
const_cast: 编译器在编译期处理，去掉类型中的const属性，主要针对const以及volatile属性。
reinterpret_cast：用来处理无关类型之间的转换。它会产生一个新的值，这个值会有与原始参数（expressoin）有完全相同的比特位。

http状态码

发表于 2016-04-11

常用状态码

100 Continue 继续，一般在发送post请求时，已发送了http header之后服务端将返回此信息，表示确认，之后发送具体参数信息
200 OK 正常返回信息
201 Created 请求成功并且服务器创建了新的资源
202 Accepted 服务器已接受请求，但尚未处理
301 Moved Permanently 请求的网页已永久移动到新位置。
302 Found 临时性重定向。
303 See Other 临时性重定向，且总是使用 GET 请求新的 URI。
304 Not Modified 自从上次请求后，请求的网页未修改过。

400 Bad Request 服务器无法理解请求的格式，客户端不应当尝试再次使用相同的内容发起请求。
401 Unauthorized 请求未授权。
403 Forbidden 禁止访问。
404 Not Found 找不到如何与 URI 相匹配的资源。

500 Internal Server Error 最常见的服务器端错误。
503 Service Unavailable 服务器端暂时无法处理请求（可能是过载或维护）。
