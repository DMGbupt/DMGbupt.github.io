linux下core文件调试

发表于 2016-07-09   |  

linux下程序异常退出时，会在当前工作目录下生成一个core文件，记录内存转储的信息，有助于debug。可以用gdb调试core文件。

  • 开启core文件的生成
    ulimit -c命令可以查看core文件的大小，若为0，说明内核不生成core文件。
    可以使用ulimit -c size设置core文件大小为size，或者使用ulimit -c unlimited不限制生成core文件大小。
  • 设置core文件名格式
    若不设置core文件的扩展名，则会全部命名为core，新的core会覆盖旧的core。
    在/proc/sys/kernel路径下的以core_开头的文件可以设置core文件的扩展名。
    比较常用的是core_uses_pid文件。文件内容为0表示生成的core文件都命名为core；内容为1表示添加进程pid作为扩展名，生成的格式为core.pid。更多的格式可以在core_pattern中设置。
  • 调试core文件
    可以使用gdb调试core文件（编译时需要加入-g选项，不然gdb无法查找变量符号，只会显示类型信息），命令gdb program core_file。
    在gdb界面，可以使用bt查看程序出错时的堆栈信息。

进程和线程区别

发表于 2016-06-17   |  

进程是程序的一次执行。进程间是独立的，表现在内存空间、上下文环境上；线程运行在进程空间内，同一进程产生的线程共享同一内存空间，当进程退出时该进程所产生的线程都会被强制退出并清除。
线程占用资源少于进程占用资源。进程拥有一个完整的虚拟地址空间，不依赖于线程而独立存在；反之，线程是进程的一部分，没有自己的地址空间，与进程内的其他线程一起共享分配给该进程的所有资源。
一个没有线程的进程可以被看作是单线程的。线程有时又被称为轻权进程或轻量级进程，也是 CPU 调度的一个基本单位。

C++ STL容器选择

发表于 2016-05-13   |  

STL（标准模板库）主要由以下部分组成：容器（vector，list，deque，set，map）、容器适配器（stack，queue，priority_queue），迭代器（五种迭代器），算法，函数对象（functor），lambda等。
由于各种容器都有各自优缺点，所以选择适当容器会事半功倍。

顺序容器

vector拥有一个连续的内存区块，支持随机访问，允许尾端的插入删除。如果需要高效的随机访问，而不在乎插入删除的效率，使用vector。当vector元素满了重新分配内存时，迭代器会失效。
如果经常在序列头、尾插入删除元素，可以采用deque。deque使用多个内存区块，支持随机访问。元素被移除时，deque会自动缩减内部内存用量，因此插入删除总会使迭代器失效。
当插入删除常发生在序列中段时，可以使用list。list是双向链表结构，拥有一段不连续的内存空间，不支持随机访问。但是只要元素还在list中，那么指向那些元素的迭代器就不会失效。

关联容器

set和map基于红黑树实现，元素是有序的。如果想允许重复的key，可以使用multiset和multimap。
unordered_set和unordered_map是基于哈希表实现的，元素是无序的。如果想允许重复的key，可以使用unordered_multiset和unordered_multimap。典型的字典结构可以使用unordered_multimap。

容器适配器

准确的说，这些适配器并没有贡献新的容器，只是在现有容器上加了一层封装，形成更统一的抽象操作，使上层不用再关注不同容器的操作函数差异。
stack，完成堆栈抽象。从<stack>头文件的定义里可以看出stack底层默认容器是deque，当然也能用vector和list，实际上通常vector更好。

queue，完成队列抽象。看下<queue>头文件里的定义，底层默认也是deque，因为队列本身就是头尾操作，很符合deque容器。也可以使用list。

priority_queue，优先级队列，也是在<queue>中定义。底层默认是vector，因为用到了stl的堆排序，需要底层容器支持随机访问。也可以使用deque，但是list不行。

指针与引用区别

发表于 2016-05-09   |  

1、从现象上看：指针在运行时可以改变其所指向的值，而引用一旦和某个对象绑定后就不再改变。
2、从内存分配上看：程序为指针变量分配内存区域，而引用不分配内存区域。
3、从编译上看：程序在编译时分别将指针和引用添加到符号表上，符号表上记录的是变量名及变量所对应地址。指针变量在符号表上对应的地址值为指针变量的地址值，而引用在符号表上对应的地址值为引用对象的地址值。符号表生成后就不会再改，因此指针可以改变指向的对象（指针变量中的值可以改），而引用对象不能改。

通俗：
引用：一个变量的别名，为什么引入别名呢？原因是我们想定义一个变量，他共享另一个变量的内存空间，使用别名无疑是一个好的选择。变量是什么？是一个内存空间的名字，如果我们给这个内存空间在起另外一个名字，那就是能够共享这个内存了，引用（别名）的由此而来。
指针：指向另一个内存空间的变量，我们可以通过它来索引另一个内存空间的内容，本身有自己的内存空间。
